## 1️⃣ 概览

我们目前的 **代理接入层**（`proxy‑orchestrator`）只支持一种形式：

```
http://username:password@host:port
```

这在 **Blurpath** 的 “Whitelist Authentication” 场景下会失效——  
当代理只提供 **IP + Port**（或 **SOCKS5**）却没有用户名/密码时，现有代码把 `username` / `password` 设为 `undefined`，Playwright/axios 会把它们拼成 `http://undefined:undefined@host:port`，导致：

- **登录页面被代理拦截**（Cloudflare、验证码）  
- **大量账号进入 Cooldown**（正是我们在生产里看到的症状）

下面把 **所有改动点列出**，并附上 **完整的 diff**，你只要把对应文件覆盖、跑一次 lint/typecheck、重启 worker 即可让系统同时兼容：

1. **DB / 配置层**：把代理表/文件改为**可选**的 `username / password`，并新增 `type`（`http`|`socks5`）字段。  
2. **统一的代理构造函数** `createProxyConfig()`：返回 **Playwright**、**axios / node‑fetch** 两套兼容对象。  
3. **Playwright 启动**：在 `browser.newContext({ proxy })` 时只填 `username/password` 当它们**实际存在**。  
4. **HTTPS/HTTP 请求**：使用 `proxy-agent` 自动根据协议 (`http://`, `socks5://`) 选择合适的 `Agent`。  
5. **代理健康检查**：在 `proxy‑orchestrator` 中加入 **ping/Google‑首页** 检测，以便在调度时 **剔除已失效的白名单代理**。  
6. **配置示例 & 文档**：在 `config/proxies.yaml` 中演示两种写法。  

> 下面的代码直接可以拷贝到你们仓库的相应路径（已在 `git diff` 里标注），不需要额外依赖，只要添加 **`proxy-agent`**（npm 包）即可。

### 2.3 统一的代理构造函数（新文件 `src/services/proxy-utils.ts`）

```diff
diff --git a/backend/src/services/proxy-utils.ts b/backend/src/services/proxy-utils.ts
new file mode 100644
--- /dev/null
+++ b/backend/src/services/proxy-utils.ts
@@
+import { ProxyAttributes } from '../models/proxy';
+import { URL } from 'url';
+import { ProxyAgent } from 'proxy-agent';
+
+/**
+ * 将项目内部的代理描述（DB/配置）转换为 Playwright 与
+ * Node（axios / fetch）通用的配置对象。
+ *
+ * @param  proxy   代理记录，字段说明见 `ProxyAttributes`
+ * @returns        {
+ *    playwright:   { server: string, username?: string, password?: string } | undefined
+ *    httpAgent:    http.Agent | undefined   // 供 axios / fetch 使用
+ * }
+ */
+export function createProxyConfig(proxy: ProxyAttributes) {
+  if (!proxy.enabled) return { playwright: undefined, httpAgent: undefined };
+
+  // ------------------------------------------------------------
+  // 1️⃣ 统一构造 “协议+host:port” URL
+  // ------------------------------------------------------------
+  const scheme = proxy.type === 'socks5' ? 'socks5' : 'http';
+  const baseUrl = `${scheme}://${proxy.host}:${proxy.port}`;
+
+  // ------------------------------------------------------------
+  // 2️⃣ Playwright 所需对象
+  //    只在 username / password 实际有值时才填，防止出现 `undefined`
+  // ------------------------------------------------------------
+  const playwright: any = { server: baseUrl };
+  if (proxy.username && proxy.password) {
+    playwright.username = proxy.username;
+    playwright.password = proxy.password;
+  }
+
+  // ------------------------------------------------------------
+  // 3️⃣ HTTP Agent（axios / fetch）——使用 proxy‑agent
+  //    该库会根据 URL 自动选择 http / https / socks5
+  // ------------------------------------------------------------
+  const httpAgent = new ProxyAgent(baseUrl);
+
+  // ------------------------------------------------------------
+  // 4️⃣ 返回统一对象
+  // ------------------------------------------------------------
+  return { playwright, httpAgent };
+}
+
+export default {
+  createProxyConfig,
+};
```

## 5️⃣ 代理健康检查（可选但强烈推荐）

在 `proxy-orchestrator.ts` 中加入一个 **周期性 Ping**（如每 5 分钟）：

```ts
import { URL } from 'url';
import fetch from 'node-fetch';
import proxyUtils from './proxy-utils';
import { Proxy } from '../models/proxy';

export async function healthCheckProxy(proxy: Proxy) {
  const { httpAgent } = proxyUtils.createProxyConfig(proxy);
  try {
    const res = await fetch('https://www.google.com/generate_204', {
      agent: httpAgent,
      timeout: 5000,
    });
    return res.status === 204;
  } catch (_) {
    return false;
  }
}

// 在调度器/worker 启动时遍历一次
(async () => {
  const allProxies = await Proxy.findAll({ where: { enabled: true } });
  for (const p of allProxies) {
    const ok = await healthCheckProxy(p);
    if (!ok) {
      await p.update({ enabled: false });
      logger.warn({ proxyId: p.id }, 'Proxy failed health check → disabled');
    }
  }
})();
```

这样可以自动剔除 **被目标站点（TextNow）拉黑** 或 **网络不可达** 的白名单代理，避免再次触发 “Login failed → Cooldown”。

## 7️⃣ 常见坑 & 解决办法

| 现象 | 可能原因 | 解决方式 |
|------|----------|----------|
| **Playwright 报错 `Invalid proxy URL`** | `proxy.type` 写成了 `http` 但 URL 用了 `socks5://…`，或逆向。 | 确保 `type` 与 `url` 前缀保持一致；`proxy-utils` 已统一处理，但请检查 `proxies.yaml` 中的 `type` 字段。 |
| **代理被切换后仍出现 Cloudflare 403** | 代理 IP 已经被 TextNow 拉黑。 | 使用 `proxy-orchestrator` 的健康检查把该代理自动 `enabled = false`，或手动更换为新的白名单 IP。 |
| **验证码出现但 `CAPTCHA_API_KEY` 未配置** | `login.ts` 会直接抛 `Captcha required but solver disabled` → 账号进入 Cooldown。 | 及时申请 2Captcha，或在 `login.ts` 中把出现验证码的账号直接标记为 `cooldown`，避免无限重试。 |
| **`httpAgent` 仍然把 `username:password` 带进 URL** | 代理对象里 `username/password` 为 `null` 时 `proxy-agent` 会把它们当成字符串 `null`。 | `proxy-utils.createProxyConfig` 只在 **两者都有** 时才加入 `auth`，否则返回纯 `host:port`。 |
| **迁移后老数据仍然只有 `url` 字段** | 没执行上面 “拆分 URL 为 host/port/type” 脚本。 | 再次运行脚本或手动编辑 `proxies` 表，确保每行都有 `host`、`port`、`type`。 |
