## 1️⃣ 当前的根因一览

| 项目 | 状态 | 说明 |
|------|------|------|
| **Ready 账号** | **2**（均已绑定可用代理） | 只有这 2 个账号可以正常进入发送链路。 |
| **Cooldown 账号** | **28** | 这些账号在登录阶段被 `Username input field not found` 直接判定为登录失败 → 进入 Cooldown → 任务回滚 → 再次被调度时仍然失败。 |
| **Pending 任务** | **36** | 全部因为前面 28+2 = 30 个账号（其中 28 在 Cooldown，2 在 Ready）而卡住。 |
| **日志关键** | `Username input field not found` → `Login failed` → `账号被打到 Cooldown` | 登录页面的 **DOM 结构 / 代理拦截 / 验证码** 已经与我们原来的选择器不匹配。 |
| **基础设施** | ✅ API、MySQL、Redis、worker、scheduler 正常运行 | 不是系统宕机，而是业务层的登录自动化失效。 |
| **Captcha** | `No API key found for 2captcha` → **Captcha 失效** | 一旦登录页面出现验证码，自动化会直接报错并把账号送进 Cooldown。 |

> **结论**：我们需要 **（1）快速定位当前登录页面实际的用户名输入框 selector**，**（2）在代码里加入容错、调试信息、fallback selector**，以及 **（3）为出现验证码的情况提供 2Captcha 支持**，**（4）检查并恢复被误判为 Cooldown 的账号**。  

下面给出 **一步步的操作方案**，每一步都有 **命令/代码示例**，你可以直接拷贝到服务器上执行。

## 2️⃣ 快速定位 **Username** 输入框（调试脚本）

> 目标：让 Playwright 在出现错误时 **保存完整的 HTML、截图以及可搜索的片段**，从而手动找出真实的 selector。

### 2.1 创建调试脚本 `debug-textnow-login.mjs`

```js
#!/usr/bin/env node
// debug-textnow-login.mjs
import { chromium } from 'playwright';
import yaml from 'js-yaml';
import fs from 'fs';
import path from 'path';
import { program } from 'commander';
import logger from '../src/logger';   // 按你项目的路径自行修改

program
  .requiredOption('-a, --account <id>', 'Account ID in DB')
  .option('-p, --proxy <url>', 'Optional proxy to override account proxy')
  .parse();

const { account: ACCOUNT_ID, proxy: PROXY_ARG } = program.opts();
const CONFIG = yaml.load(fs.readFileSync(
  '/var/www/massmail/backend/src/config/textnow.yaml',
  'utf8'
));
const DB = /* 这里请自行初始化你的 DB 连接，或直接 `require` 现有模型 */ null;

// ------------ 读取账号信息 ------------
async function getAccount(id) {
  // 调整为你项目中获取 account 的方式
  const row = await DB('accounts').where('id', id).first();
  if (!row) throw new Error(`Account ${id} not found`);
  return row;
}

// ------------ 主函数 ------------
(async () => {
  const acc = await getAccount(ACCOUNT_ID);
  const proxyUrl = PROXY_ARG || acc.proxy_url; // 你的 schema 中的字段名

  const browser = await chromium.launch({
    headless: false, // 打开 UI，手动调试更方便
    args: proxyUrl ? [`--proxy-server=${proxyUrl}`] : [],
  });
  const context = await browser.newContext();
  const page = await context.newPage();

  try {
    console.log(`🔎 正在打开 ${CONFIG.textnow.loginUrl}`);
    await page.goto(CONFIG.textnow.loginUrl, { waitUntil: 'networkidle', timeout: 30000 });

    // 保存完整页面 HTML
    const html = await page.content();
    const htmlPath = `/tmp/debug_account${ACCOUNT_ID}_html_${Date.now()}.html`;
    fs.writeFileSync(htmlPath, html);
    console.log(`🗂 HTML 已保存 → ${htmlPath}`);

    // 保存截图
    const shotPath = `/tmp/debug_account${ACCOUNT_ID}_screenshot_${Date.now()}.png`;
    await page.screenshot({ path: shotPath, fullPage: true });
    console.log(`📸 截图已保存 → ${shotPath}`);

    // 尝试几个常见的 selector，记录是否匹配
    const candidates = [
      'input[name="username"]',
      'input[name="email"]',
      'input[type="email"]',
      'input#login-email',
      'input#email',
      'input[data-testid="email"]',
      'input[placeholder*="email"]',
      'input[placeholder*="username"]',
    ];
    for (const sel of candidates) {
      const el = await page.$(sel);
      console.log(`🔎 selector "${sel}" → ${el ? 'FOUND' : 'NOT FOUND'}`);
    }

    // 如果都未找到，打印页面上所有的 input 元素供手动筛选
    if (!candidates.some(sel => page.$(sel))) {
      const inputs = await page.$$eval('input', nodes =>
        nodes.map(n => ({
          type: n.type,
          name: n.name,
          id: n.id,
          placeholder: n.placeholder,
          outerHTML: n.outerHTML,
        }))
      );
      const outPath = `/tmp/debug_account${ACCOUNT_ID}_inputs_${Date.now()}.json`;
      fs.writeFileSync(outPath, JSON.stringify(inputs, null, 2));
      console.log(`ℹ️ 所有 <input> 元素已导出 → ${outPath}`);
    }

    // 检查是否出现 Cloudflare / 重新定向页面
    if (/blocked|access denied|cf\-ray|challenge/i.test(html)) {
      console.warn('⚠️ 页面似乎被 Cloudflare / 代理拦截，请检查代理 IP');
    }

    // 检测是否出现验证码 iframe
    const cfFrame = page.frames().find(f => /recaptcha|captcha|challenge/i.test(f.url()));
    if (cfFrame) {
      console.warn('🧩 检测到验证码 iframe，若需要自动求解，请配置 2CAPTCHA KEY');
    }
  } catch (e) {
    console.error('❌ 调试脚本异常：', e);
  } finally {
    await browser.close();
  }
})();
```

## 3️⃣ 更新登录代码：加入调试、容错、fallback selector + Captcha

> 这里采用 **“配置化 + 多 selector”** 的思路，让以后 UI 变化只需要改 `textnow.yaml`，而不必改代码。

### 3.1 `backend/src/config/textnow.yaml`（示例）

```yaml
textnow:
  loginUrl: "https://app.textnow.com/login"

  # 多个候选 selector，按顺序尝试；如果前面的找不到则尝试下一个
  selectors:
    username:
      - 'input[name="username"]'
      - 'input[name="email"]'
      - 'input[type="email"]'
      - 'input#login-email'
      - 'input#email'
      - 'input[placeholder*="email"]'
      - 'input[placeholder*="username"]'
    password:
      - 'input[name="password"]'
      - 'input#login-pass'
      - 'input[type="password"]'
      - 'input#password'
    submit:
      - 'button[type="submit"]'
      - 'button[data-action="login"]'
      - 'button#login-button'
    loggedInIndicator:
      - 'div[data-testid="inbox"]'         # 已登录后出现的页面标记
      - 'a[data-testid="profile-link"]'
      - 'nav[aria-label="Main navigation"]' # 任选其一，只要唯一即可
    countryModalClose:
      - 'button[data-action="close"]'
      - 'button#close-modal'
    captchaFrame:
      - '.*(recaptcha|2captcha|challenge|cf_chl).*'   # 正则匹配 iframe URL
```

> **重点**：**`selectors.username` 现在是一个数组**，代码会依次尝试，直到找到可见元素；如果全部不匹配则抛 `Username input field not found` 并记录调试截图（见下文）。

### 3.2 更新 `backend/src/services/textnow/login.ts`

```ts
// backend/src/services/textnow/login.ts
import type { Page } from 'playwright';
import logger from '../../logger';
import type { Account, Proxy } from '../../models';
import yaml from 'js-yaml';
import fs from 'fs';
import { solveCaptcha } from '../../captcha-solver';

// 读取一次配置（生产环境里可缓存，避免每次 IO）
const cfg = yaml.load(
  '/var/www/massmail/backend/src/config/textnow.yaml',
  'utf8'
) as any;

async function pickFirstVisible(pageOrFrame: Page | any, selectors: string[]): Promise<any> {
  for (const sel of selectors) {
    try {
      // 若 selector 本身是正则（用于 iframe）则跳过此函数
      const element = await pageOrFrame.waitForSelector(sel, {
        state: 'visible',
        timeout: 5000,
      });
      if (element) return element;
    } catch {
      // ignore – try next selector
    }
  }
  return null;
}

export async function login(page: Page, account: Account, proxy?: Proxy): Promise<void> {
  // 1️⃣ 打开登录页 + 截图（失败时也会留痕）
  try {
    await page.goto(cfg.textnow.loginUrl, { waitUntil: 'networkidle', timeout: 30000 });
    await page.screenshot({ path: `/tmp/${account.id}_login_start.png` });
  } catch (e) {
    logger.error({ accountId: account.id, err: e }, 'Login page load failed');
    throw e;
  }

  // 2️⃣ 检查是否被 Cloudflare / 代理拦截
  const body = await page.content();
  if (/blocked|access denied|cf\-ray|challenge/i.test(body)) {
    logger.warn({ accountId: account.id, proxy }, 'Login page appears blocked');
    await page.screenshot({ path: `/tmp/${account.id}_blocked.png` });
    throw new Error('Login page blocked – likely proxy problem');
  }

  // 3️⃣ 关闭首次弹窗（如果有）
  if (await page.$(cfg.textnow.selectors.countryModalClose?.[0] ?? '')) {
    await page.click(cfg.textnow.selectors.countryModalClose?.[0]);
    logger.info({ accountId: account.id }, 'Closed country/modal dialog');
  }

  // 4️⃣ 尝试定位 **用户名**（循环 candidates）
  const usernameField = await pickFirstVisible(page, cfg.textnow.selectors.username);
  if (!usernameField) {
    await page.screenshot({ path: `/tmp/${account.id}_no_username.png` });
    logger.error(
      { accountId: account.id, candidates: cfg.textnow.selectors.username },
      'Username input field not found'
    );
    throw new Error('Username input field not found');
  }
  await usernameField.fill(account.username);

  // 5️⃣ 密码
  const passwordField = await pickFirstVisible(page, cfg.textnow.selectors.password);
  if (!passwordField) {
    await page.screenshot({ path: `/tmp/${account.id}_no_password.png` });
    logger.error({ accountId: account.id }, 'Password input field not found');
    throw new Error('Password input field not found');
  }
  await passwordField.fill(account.password);

  // 6️⃣ 验证码（如果出现）
  const captchaFrame = page.frame({ url: new RegExp(cfg.textnow.selectors.captchaFrame?.[0] ?? '') });
  if (captchaFrame) {
    if (!process.env.CAPTCHA_API_KEY) {
      logger.warn({ accountId: account.id }, 'Captcha present but 2captcha key missing');
      await page.screenshot({ path: `/tmp/${account.id}_captcha_missing_key.png` });
      throw new Error('Captcha required but solver disabled');
    }
    const token = await solveCaptcha(captchaFrame);
    const respField = await page.waitForSelector(
      'textarea[name="g-recaptcha-response"], input[name="captcha-response"]',
      { state: 'attached', timeout: 5000 }
    );
    await respField.fill(token);
    logger.info({ accountId: account.id }, 'Solved captcha via 2captcha');
  }

  // 7️⃣ 点击 **提交/登录** 按钮
  const submitBtn = await pickFirstVisible(page, cfg.textnow.selectors.submit);
  if (!submitBtn) {
    await page.screenshot({ path: `/tmp/${account.id}_no_submit.png` });
    logger.error({ accountId: account.id }, 'Submit button not found');
    throw new Error('Submit button not found');
  }
  await submitBtn.click();

  // 8️⃣ 等待页面完全加载
  await page.waitForLoadState('networkidle', { timeout: 15000 });

  // 9️⃣ 判断登录成功标记
  const successMarker = await page.$(cfg.textnow.selectors.loggedInIndicator?.[0] ?? '');
  if (successMarker) {
    logger.info({ accountId: account.id }, 'Login success');
    await page.screenshot({ path: `/tmp/${account.id}_login_success.png` });
    return;
  }

  // 10️⃣ 若仍未成功 → 记录失败截图
  await page.screenshot({ path: `/tmp/${account.id}_login_failed.png` });
  logger.error({ accountId: account.id }, 'Login failed after submit');
  throw new Error('Login failed after submit');
}
```

#### 关键改动解释

| 改动 | 目的 |
|------|------|
| **配置化多 selector** | 当 UI 改动，只改 `textnow.yaml` 中的数组，无需改代码 |
| **`pickFirstVisible`** | 依次尝试多个 selector，提升容错 |
| **失败时保存截图** (`_no_username.png`, `_login_failed.png`…) | 之后快速定位到底是哪一步卡住 |
| **检测 Cloudflare / 代理拦截** | 若代理已被拉黑，直接抛异常，防止循环进入 Cooldown |
| **验证码自动求解**（仅在 `CAPTCHA_API_KEY` 存在时） | 解决出现验证码时的卡死；如果暂时没有 key，系统会把该账号标记为 Cooldown，避免无限重试 |
| **登录成功标记** (`loggedInIndicator`) | 用页面唯一出现的元素判断成功，不再依赖 URL 变化 |

> **部署**：把上面的 `login.ts` 覆盖原文件，**在生产环境先做一次 `npm run lint && npm run typecheck`** 保证没有 TypeScript 错误；随后 **重启 worker**：  

```bash
# 假设服务名为 massmail-worker
systemctl restart massmail-worker
journalctl -u massmail-worker -f   # 实时观察是否还有 “Username input field not found”
```

## 4️⃣ **清理 / 恢复 **Cooldown** 账号**（临时快速解决）

### 4.1 手动恢复账号（如果已经确认登录可以成功）

1. **在本机（或服务器）手动使用浏览器** 用该账号 **登录 TextNow**（使用相同的代理 IP），**确保不出现验证码**。
2. 登录成功后 **复制登录后留下的 cookie**（或直接记录 `session_id`）。
3. **在 DB 中把对应账号状态改回 `ready`**，并把 `cooldown_until` 设为过去时间：

```sql
-- 假设 accounts 表结构如下
UPDATE accounts
SET status='ready', cooldown_until = NULL, last_login_at = NOW()
WHERE id = 29;   -- 你的具体账号 ID
```

4. **重新分配代理**（若该账号的 `proxy_id` 失效，换成一个健康的代理）：

```sql
UPDATE accounts
SET proxy_id = (SELECT id FROM proxies WHERE is_active = true ORDER BY RAND() LIMIT 1)
WHERE id = 29;
```

> **批量清理**：若想一次性把所有 `cooldown` 账号标记为 `ready`（仅在你已经手动确认登录成功的情况下）：

```sql
UPDATE accounts
SET status='ready', cooldown_until = NULL
WHERE status='cooldown';
```

> **注意**：如果这些账号在真正登录时仍会触发 “Username input field not found”，恢复后会再次进入 Cooldown，除非先解决 UI/selector 问题。

### 4.2 自动化 “Cooldown 恢复” 机制（可选）

在 `scheduler` / `worker` 里加入 **健康检查任务**：

- **每 30 min** 读取 `accounts` 表中 `status='cooldown' && cooldown_until < NOW() - INTERVAL '2 days'`（即已进入 2 天的 cooldown），**尝试一次轻量登录**（只输入用户名/密码，不发短信）：

  ```ts
  // pseudo: healthCheckAccount(accountId)
  const page = await browser.newPage();
  await page.goto(cfg.textnow.loginUrl);
  // 填入用户名、密码，尝试登录
  // 成功 → UPDATE accounts SET status='ready', cooldown_until=NULL;
  // 失败 → 继续保持 cooldown（或记录错误次数）
  ```

- **日志**：如果连续 3 次健康检查仍失败，自动把该账号标记为 `blocked`，不再调度。

## 5️⃣ **Captcha 失效处理**（如果你计划启用 2Captcha）

1. **申请 2Captcha / Anti‑Captcha**（或类似服务），得到 `API_KEY`。  
2. **把 key 加入系统环境变量**（在 `systemd` service 文件或 Docker Compose 中）：

```ini
# /etc/systemd/system/massmail-worker.service
Environment="CAPTCHA_API_KEY=YOUR_2CAPTCHA_KEY"
```

3. **重启 worker**：

```bash
systemctl daemon-reload
systemctl restart massmail-worker
```

4. **验证**：在 `captcha-solver.log`里应该出现 `solved captcha` 的记录，而不再出现 “No API key found”。  

> **如果暂时没有 key**，建议把 **出现验证码的账号直接转入 Cooldown**，避免无限重试。已经在新版 `login.ts` 中实现了这种行为。

## 6️⃣ **代理质量检测**（防止代理导致 Cloudflare 拦截）

在 **调度/worker** 把每个账号的代理先 **健康检测一次**：

```ts
import fetch from 'node-fetch';

async function testProxy(proxyUrl: string): Promise<boolean> {
  try {
    const res = await fetch('https://app.textnow.com/login', {
      method: 'GET',
      timeout: 8000,
      agent: new (require('https-proxy-agent'))(proxyUrl),
    });
    const text = await res.text();
    // 简单判断是否返回登录页的关键文字
    return res.ok && /login|signin/i.test(text);
  } catch (e) {
    return false;
  }
}
```

- **如果返回 `false`**，在 `accounts` 表中把对应 `proxy_id` 标记为 `bad`，并 **重新为账号挑选新的可用代理**（从 `proxies` 表里 `is_active=true` 随机挑一个）。  
- 对于已经进入 `Cooldown` 的账号，**先测试其代理**，若代理失效则先换代理再尝试恢复。

| **④ 重启 worker** | 让改动生效，观察日志是否仍出现 `Username input field not found` | 1 min |
| **⑤ 如仍报错 → 检查代理** | 用 `testProxy` 脚本快速验证代理可达性，必要时换代理 | 5 min |
| **⑥ (可选) 配置 2Captcha** | 解决验证码导致的 Cooldown（若你拥有 key） | 3 min |
| **⑦ 账号恢复** | 手动或自动把已确认可登录的账号状态改为 `ready`，重新进入调度池 | 5‑15 min（批量或单个） |

> **只要第 ①‑④ 完成**，**`Username input field not found`** 应该消失，账号会恢复 `ready`，调度器又能把任务重新派发，**Pending → Running → Sent** 的完整链路又会恢复。
